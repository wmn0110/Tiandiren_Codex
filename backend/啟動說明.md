# 後端啟動說明

這個後端是用 Flask 寫的，功能很簡單，主要是：

- 讀取所有商品
- 讀取單一商品
- 新增商品
- 讀取所有經歷
- 讀取單一經歷
- 新增經歷

資料會存在 `backend/data.json`。

## 一、安裝需要的套件

先進入 `backend` 資料夾：

```bash
cd backend
```

如果你有使用虛擬環境，先啟動它。

這次我實際驗證的 Python 環境是：

```text
C:\Users\user\anaconda3\envs\Tiandiren\python.exe
```

如果你想直接用這個環境，可以先啟用它：

```powershell
conda activate Tiandiren
```

如果你是自己建立的 `venv`，那就請改成你自己的啟動方式，例如：

```powershell
.\.venv\Scripts\Activate.ps1
```

接著安裝套件：

```bash
pip install -r requirements.txt
```

## 二、如何啟動後端

在 `backend` 資料夾中執行：

```bash
python app.py
```

如果啟動成功，你會看到類似這樣的訊息：

```text
Running on http://127.0.0.1:5000
```

## 三、如何確認後端有成功啟動

你可以用瀏覽器打開：

```text
http://127.0.0.1:5000/api/health
```

如果成功，你會看到：

```json
{
  "status": "ok",
  "message": "backend is running"
}
```

你也可以測試以下網址：

- `http://127.0.0.1:5000/api/products`
- `http://127.0.0.1:5000/api/experiences`

如果有看到 JSON 清單，就代表後端正常。

## 四、API 說明

### 商品

- `GET /api/products`
- `GET /api/products/<id>`
- `POST /api/products`

### 經歷

- `GET /api/experiences`
- `GET /api/experiences/<id>`
- `POST /api/experiences`

## 五、補充說明

- 這個後端是教學用途，程式盡量寫得簡單好懂
- 新增資料後會直接寫入 `backend/data.json`
- 重新啟動後端，資料還會在
- 如果你之後要讓前端真的串接後端，可以把前端的商品與經歷資料改成呼叫這些 API
