# 網站資料說明

這份說明是給之後要修改這個個人品牌網站的人看的。

目前這個網站還沒有串後端，所以所有資料都先放在前端裡面。
如果你之後要接後端，只要先看這份文件，就能知道現在的資料是放在哪裡、該怎麼改。

## 一、目前網站使用到的資料種類

這個網站主要有三種資料：

### 1. 個人基本資料

包含：

- 姓名
- 職稱
- 簡介
- Email
- 社群連結

這些資料目前放在：

- `frontend/src/data/siteData.js`

### 2. 商品資料

每一筆商品會包含：

- 商品名稱
- 商品圖片
- 商品簡介
- 商品詳細介紹
- 適合對象
- 價格或合作方式

這些資料目前放在：

- `frontend/src/data/siteData.js`

### 3. 經歷資料

每一筆經歷會包含：

- 經歷標題
- 經歷圖片
- 經歷簡介
- 詳細故事
- 日期或年份
- 相關成果

這些資料目前放在：

- `frontend/src/data/siteData.js`

---

## 二、商品資料要怎麼修改

如果你要修改商品內容，請打開：

- `frontend/src/data/siteData.js`

找到這一段：

```js
export const products = [
  ...
];
```

裡面的每一個物件代表一個商品。

例如：

```js
{
  id: 1,
  name: '高效能桌機組合',
  image: product1Image,
  intro: '兼顧穩定性與多工效率的實用配置。',
  detail: '更完整的商品介紹...',
  audience: '學生、工程師、內容創作者',
  pricing: 'NT$ 28,000 起，可依預算客製規格',
}
```

你可以直接改這些欄位：

- `name`：商品名稱
- `image`：商品圖片
- `intro`：商品簡介
- `detail`：商品詳細介紹
- `audience`：適合對象
- `pricing`：價格或合作方式

### 修改提醒

- 如果你新增商品，請照著現有格式再新增一個物件
- 如果你刪掉商品，前端畫面就會少一張卡片
- `id` 建議保持每筆都不同

---

## 三、經歷資料要怎麼修改

如果你要修改經歷內容，請打開：

- `frontend/src/data/siteData.js`

找到這一段：

```js
export const experiences = [
  ...
];
```

裡面的每一個物件代表一段經歷。

例如：

```js
{
  id: 1,
  title: '學生時期的基礎累積',
  image: experience1Image,
  intro: '從課堂、筆記與專題開始建立學習節奏。',
  story: '更完整的經歷故事...',
  date: '2019 - 2022',
  result: '完成課堂專題、建立 Git 使用習慣、累積自學紀錄',
}
```

你可以直接改這些欄位：

- `title`：經歷標題
- `image`：經歷圖片
- `intro`：經歷簡介
- `story`：詳細故事
- `date`：日期或年份
- `result`：相關成果

---

## 四、圖片要放在哪裡

圖片建議放在以下兩個資料夾：

- `frontend/商品圖片`
- `frontend/經歷圖片`

目前網站就是這樣整理的。

### 商品圖片

請把商品圖片放在：

- `frontend/商品圖片/product-1.jpg`
- `frontend/商品圖片/product-2.jpg`
- `frontend/商品圖片/product-3.jpg`

### 經歷圖片

請把經歷圖片放在：

- `frontend/經歷圖片/experience-1.jpg`
- `frontend/經歷圖片/experience-2.jpg`
- `frontend/經歷圖片/experience-3.jpg`

---

## 五、圖片名稱應該怎麼命名

建議使用這種簡單、固定的命名方式：

- 商品圖片：`product-1.jpg`、`product-2.jpg`、`product-3.jpg`
- 經歷圖片：`experience-1.jpg`、`experience-2.jpg`、`experience-3.jpg`

### 為什麼這樣命名比較好

- 容易看懂
- 容易對應資料
- 之後要接後端時也比較好整理
- 不容易和其他圖片混淆

### 命名建議

- 盡量全部使用英文小寫
- 中間用 `-` 分隔
- 不要用太長的檔名
- 不要有空白字元

例如：

- `product-1.jpg`
- `product-2.jpg`
- `experience-1.jpg`

---

## 六、目前前端資料放在哪裡

目前最重要的資料檔是：

- `frontend/src/data/siteData.js`

這個檔案已經把：

- 個人基本資料
- 商品資料
- 經歷資料

都集中在一起了。

這樣的好處是：

- 之後要修改內容比較快
- 之後要改成後端資料時，也比較容易替換
- 初學者只要看一個檔案就知道資料在哪裡

---

## 七、如果之後要接後端，建議怎麼做

如果你之後想把資料改成後端提供，最適合先替換的地方就是：

- `frontend/src/data/siteData.js`

因為現在畫面元件只負責顯示資料，資料本身是獨立放在這個檔案裡。

未來可以把這些靜態資料改成：

- 從 API 取得
- 從資料庫讀取
- 從管理介面新增或修改

---

## 八、給初學者的修改方式

如果你只是想改網站內容，最簡單的做法是：

1. 打開 `frontend/src/data/siteData.js`
2. 找到你要改的資料
3. 直接修改文字或圖片名稱
4. 存檔後重新整理網頁

如果你改的是圖片：

1. 把新圖片放進 `frontend/商品圖片` 或 `frontend/經歷圖片`
2. 回到 `frontend/src/data/siteData.js`
3. 把對應的圖片匯入名稱一起改掉

---

## 九、總結

目前這個網站已經把資料整理成一個比較容易維護的形式。

你現在如果要改內容，請先看：

- `frontend/src/data/siteData.js`

如果你要改圖片，請先放到：

- `frontend/商品圖片`
- `frontend/經歷圖片`

這樣之後要接後端資料管理功能時，也會比較順。
